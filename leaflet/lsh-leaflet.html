LSH

Local Smart Home — Unified Automation Dashboard

Self-hosted · Open Platform

Works fully offline · Zero cloud dependency · Apple HomeKit native

One dashboard. Every device. No cloud required.

LSH is a self-hosted Node.js platform that unifies Victron Energy, KNX bus, IP cameras, HVAC, lighting, alarms, and 20+ smart home ecosystems into a single, real-time web interface — with native Apple HomeKit bridge, on-demand RTSP streaming, and full REST API included.

🔒

100 % local by default

All data stays inside your LAN. MQTT and Modbus run directly — no relay server, no account required. Optional VRM cloud fallback for remote access when you need it.

🔌

Protocol-agnostic architecture

MQTT, KNXnet/IP, Modbus TCP, REST, WebSocket, RTSP — add any device using standard protocols. New integrations are pluggable Node.js modules.

🍎

Native Apple HomeKit

Every sensor, switch, camera, thermostat, and relay is automatically exposed to the Apple Home app. Control your entire home from Siri or your iPhone.

Victron Energy Hub

Deep integration with Victron CCGX/Venus OS via local MQTT. Monitor solar MPPT chargers, battery bank SoC, grid import/export, AC/DC loads, and inverter state in real time. Relay outputs controlled directly from the dashboard — trigger based on SoC thresholds or manual override.

  • Live kW/kWh metrics with trend arrows
  • MQTT → VRM cloud automatic failover
  • SolarEdge production data overlay
📷

Camera & Video

UniFi Protect cameras are automatically discovered and displayed as live thumbnails. Any RTSP camera (Hikvision, Dahua, Reolink…) can be re-streamed on-demand via the built-in FFmpeg RTSP proxy — each camera gets its own port, stream starts only when a client connects.

  • On-demand FFmpeg re-streaming (no idle bandwidth)
  • Loxone Intercom & camera app compatible
  • UniFi Protect motion event push
🏠

Rooms & Sensors

Group sensors, lights, and switches into rooms with a drag-and-drop editor. Temperature, humidity, CO₂, motion, occupancy, door/window contacts, and air quality all appear in a unified timeline. Room cards update in real time via Socket.IO — no page refresh needed.

  • Room modal editor — assign any sensor to any room
  • Live value badges with color-coded thresholds
  • Historical mini-charts for temperature / humidity
🔆

Lighting & Switches

Control IKEA, Shelly, Zigbee, KNX, ESPHome, and BoneIO lights and switches from a single panel. Dimmers, color temperature, RGB, scenes — all surfaced as HomeKit accessories automatically. Tactile feedback and instant state sync via MQTT or native API.

  • Brightness + color temperature sliders
  • SmartThings, Dirigera & Trådfri supported
  • BroadLink IR/RF scene macros
📡

MQTT Explorer

Built-in live MQTT topic browser with subscribe and publish panels. Inspect any topic value as it updates, send test commands, and onboard custom DIY devices without leaving the dashboard. Separate connection from the Victron bus — safe to use simultaneously.

  • Topic tree with live value display
  • Retained message inspection
  • SmartBob & DIY MQTT device support
🔐

Security & Access

Built-in user management with bcrypt password hashing. Automatic Let's Encrypt TLS with hot certificate renewal — no restarts. Satel Integra alarm panel integration for zone status, arm/disarm, and sensor display. Optional HTTP → HTTPS redirect on the standard port.

  • Let's Encrypt auto-renew with ACME HTTP-01
  • Manual cert / self-signed TLS supported
  • Satel Integra alarm zones & partitions

Technical Reference — REST API, Config, Deployment

Developer Reference

REST API · HomeKit · TLS · Config schema

Getting started in 3 steps
1

Clone & Configure

Copy config.example.json → config.json. Fill in your MQTT broker address and/or VRM credentials. Set your HomeKit PIN. Run node server.js — live data appears within seconds.

2

Connect Integrations

Open /settings.html in your browser. Each integration has a dedicated card with a one-click connectivity test. No config file editing — fill in the form and click Save. New devices appear in the dashboard immediately.

3

Add to Apple Home

Open the HomeKit QR code from Settings. Scan with your iPhone — all connected devices instantly appear in Apple Home. Control them via Siri, automations, or the Home app. Bridge survives server restarts — no re-pairing.

Minimal config.json
{ "mqtt": { "host": "192.168.1.109", "portalId": "abc123" }, "vrm": { "email": "user@email.com", "installationId": 12345 }, "knx": { "host": "192.168.1.50", "groupAddresses": [ { "address": "1/1/1", "dpt": "DPT9", "label": "Living Temp" } ] }, "ffmpegRtsp": { "enabled": true, "basePort": 8554 }, "homekit": { "pin": "031-45-154" }, "server": { "port": 3001, "letsEncrypt": { "enabled": true, "domain": "lsh.yourdomain.com" } } }

Deployment Options

  • Bare metal Raspberry Pi 4 running Raspberry Pi OS Lite — recommended for energy monitoring close to Victron hardware
  • Docker Single container — mount config.json and persist/ as volumes; expose ports 3001 + HomeKit HAP port
  • NAS / VM Synology DSM, Proxmox LXC, TrueNAS Jail — any Linux environment with Node.js and network access to MQTT broker
  • systemd service Auto-start on boot, journal logging, automatic restart on crash — one-command service install included
🔮

What Makes LSH Different

  • No Home Assistant dependency — standalone Node.js, minimal attack surface
  • Victron-first — designed for off-grid / solar installations, not just consumer IoT
  • MQTT bus explorer built in — no separate MQTT.fx or MQTT Explorer needed
  • On-demand RTSP proxy — no bandwidth wasted when no one is watching cameras
  • KNX bus native — first-class KNXnet/IP support with per-address DPT config
  • Settings UI — add devices through browser forms, not YAML or Python scripts
  • SIP softphone — browser-based VoIP so you can answer the door intercom from the dashboard

Strategic Analysis — SWOT

Business Analysis

Strengths · Weaknesses · Opportunities · Threats

💪
Strengths
Internal · Positive
✦
Local-first, zero cloud dependency
All data stays on LAN. MQTT runs directly to hardware — no relay server, no account, works during internet outages.
✦
20+ integrations in a single process
~120 MB RAM footprint. No microservice sprawl, no Docker Compose with 12 containers — just node server.js.
✦
Victron Energy depth
Solar, battery SoC, grid, relay control — uniquely positioned for off-grid and solar installations. SolarEdge overlay included.
✦
Native Apple HomeKit bridge
Every sensor, switch, and camera auto-exposed. Unlimited accessories — survives restarts without re-pairing.
✦
Protocol breadth
MQTT, KNXnet/IP, Modbus TCP, REST, WebSocket, RTSP — standard protocols only, no proprietary SDKs.
✦
Settings UI — no YAML required
All integrations configured via browser forms with live connectivity tests. KNX group addresses, SmartBob topics, FFmpeg proxy — all point-and-click.
⚠️
Weaknesses
Internal · Negative
▲
Technical setup required
No one-click installer or Docker Hub image. Requires Node.js, terminal comfort, and manual config.json editing for initial setup.
▲
No automation engine
No built-in rule or scene builder. Users must rely on external tools (Node-RED, cron, HomeKit automations) for time or condition-based triggers.
▲
No persistent history / charting
Live values only — no time-series database, no energy graphs over days or weeks. Historical data requires InfluxDB + Grafana sidecar.
▲
Single-node, no HA failover
One server failure = no dashboard, no HomeKit bridge. No clustering or standby replication supported.
▲
Cloud API fragility
SmartThings, LG ThinQ, and VRM integrations depend on vendor API stability. Unilateral breaking changes can silently disable features.
▲
Single maintainer bus factor
Platform continuity depends on one developer. No community governance, no contributor pipeline, no public issue tracker yet.
🚀
Opportunities
External · Positive
◆
Privacy-conscious market growth
Users increasingly reject cloud-dependent hubs after outages (SmartThings 2022, Google Nest 2023). LSH's local-first story is a direct answer.
◆
Victron / solar / EV adoption rising
Victron Energy is the dominant brand in off-grid solar. EV charger integration + battery relay control gives LSH a unique energy management angle no competitor covers.
◆
KNX building automation market
KNX is the dominant bus in European commercial buildings. First-class KNXnet/IP support opens the professional integrator and architect market.
◆
Docker image + NAS packaging
A published Docker Hub image and Synology/Unraid community package would unlock a large segment of homelab users who deploy from package managers.
◆
Commercial integrator channel
KNX + Victron + Loxone overlap with professional AV and building automation installers — a channel underserved by Home Assistant's DIY-first positioning.
◆
Automation engine add-on
Adding a Node-RED-style visual flow builder or a simple rule engine (if sensor X > value → command Y) would significantly expand the non-technical user segment.
🔴
Threats
External · Negative
●
Home Assistant dominance
Enormous ecosystem (3,000+ integrations), massive community, strong brand, and well-funded Nabu Casa behind it. Mindshare is very hard to displace.
●
Vendor API changes
SmartThings, LG ThinQ, and VRM can revoke or break APIs without notice. No contractual SLA between LSH and any vendor platform.
●
Matter / Thread commoditisation
If Matter becomes universal, Apple Home and Google Home absorb device control natively — reducing the differentiation of multi-protocol middleware like LSH.
●
Security exposure at network edge
Self-hosted dashboards port-forwarded to the internet are a common attack vector. Misconfigured TLS or weak passwords can expose the entire home network.
●
Competing open platforms
openHAB, Domoticz, ioBroker, and Gladys cover overlapping protocol breadth with established communities and package registries.
●
HAP-nodejs / HomeKit spec changes
The HAP bridge relies on a reverse-engineered Apple spec. Apple can introduce MFi restrictions or firmware changes that break the bridge without warning.
