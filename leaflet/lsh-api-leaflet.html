REST + WebSocket
One API surface for every protocol in your building

One API. Every Device. Zero Lock-in.

LSH ingests 15+ home automation protocols — MQTT, WebSocket, REST, CoAP, UDP, cloud APIs — normalises them into a single live data store, and exposes everything through one consistent REST + WebSocket interface. Read any sensor. Send any command. Subscribe to all updates. One token.

📡

Single endpoint

GET /api/devices returns every device from every integration with live values — no per-vendor SDK required.

⚡

Universal write

POST /api/device/:key/command works the same for SmartThings, Fibaro, Loxone, KNX, Shelly, AuxAir, and 10 more.

🔄

Real-time push

One Socket.IO connection delivers live updates from all integrations — no polling, no per-vendor WebSocket.

Real-time push — Socket.IO
// Browser / Node.js const socket = io('https://lsh.local:3001', { auth: { token: 'Bearer <token>' } }) // All integrations — one event socket.on('state', ({ key, value }) => { // e.g. key = "auxair/12345/temp" // value = 22.5 updateUI(key, value) }) // Sensor registry changes socket.on('devices', (devices) => { renderDeviceList(devices) })
📡

Sub-second latency

Somfy events polled every 1 s. Bayrol MQTT pushed on change. Victron MQTT is live. All arrive on the same Socket.IO stream.

🔁

No polling required

LSH handles all protocol-level polling and push internally. Consumers just listen — no reconnect logic, no per-vendor heartbeat.
